# I Built an Automated AI Morning Briefing System with Claude Cowork, Notion, and NotebookLM

Every morning, before I even open my eyes, a fully-researched, formatted, audio-enabled briefing is waiting for me in Notion. Eight sections of sourced content. Live article links. A ~10-minute AI-generated podcast summarizing the whole thing. Zero manual work.

This is how I built it — and how you can too.

---

## What It Does

The system runs on a schedule — multiple times per day — and delivers a "Morning Briefing" page directly into a Notion database. Each briefing includes:

- **8 researched content sections** with sourced bullet points and live links
- **A podcast-style audio overview** generated by NotebookLM
- **Automatic run numbering** (e.g., Briefing #16 = the 16th briefing ever delivered)
- **Time slot detection** — Morning, Afternoon, or Evening based on when it runs
- **A persona named Lera** — a sharp, witty AI briefing assistant with dry humor and zero corporate-speak

No tabs. No scrolling. No "let me check the news real quick" rabbit holes. Just a clean, sourced briefing ready when I am.

---

## The Stack

| Tool | Role |
|------|------|
| **Claude Cowork** (desktop app) | Orchestrates the entire pipeline as a scheduled task |
| **Notion** (via Notion MCP) | Stores and displays each briefing as a database entry |
| **NotebookLM** (via Claude-in-Chrome MCP) | Generates a ~10-minute audio podcast from briefing content |
| **WebSearch** | Live web research across all 8 content categories |

The magic is in how these connect. Claude Cowork acts as the orchestration layer — it triggers the pipeline on a cron schedule, calls the Notion MCP to manage database entries, runs web searches for live content, and drives Chrome automation to interact with NotebookLM. No custom backend. No API glue code. Just MCP tools wired together through a skill definition.

## The Pipeline — Step by Step

Here's exactly what happens each time the pipeline fires:

### Step 1: Determine the Run Number

The pipeline queries the Notion Morning Briefing database to count existing entries and determine the next sequential run number. Briefing #16 means 15 briefings came before it.

### Step 2: Detect the Time Slot

Based on the current time, the system tags the briefing as **Morning**, **Afternoon**, or **Evening**. This drives both the title metadata and the tone Lera uses in the intro.

### Step 3: Research 8 Categories via Web Search

This is the meat of the pipeline. Live web searches run across eight categories:

1. **Weather** — local forecast pulled by ZIP code
2. **World News** — top headlines and breaking stories
3. **K-12 Education** — policy updates, EdTech, and classroom trends
4. **Tech** — developer tools, product launches, industry shifts
5. **Claude AI News** — the latest from Anthropic
6. **Claude Code / Cowork Updates** — new features, releases, community builds
7. **Cybersecurity** — threats, vulnerabilities, and advisories
8. **Trending X Hashtags** — what's dominating the conversation right now

Each section gets 3-5 sourced bullet points with live article links. No hallucinated URLs. Every link comes from real web search results.

### Step 4: Create the Notion Page

A fully formatted briefing page gets created in Notion with:

- **Title:** `YYYY-MM-DD — Morning Briefing #N`
- **Properties:** Run Number (integer), Status (Published), Time Slot, Date, Tags (multi-select)
- **Content:** All 8 sections with emoji headers, sourced bullets, and links
- **Footer:** A motivational quote and the run number stamp

### Step 5: Generate Audio via NotebookLM

This is where it gets wild. Using Claude-in-Chrome MCP browser automation, the pipeline:

1. Opens NotebookLM in Chrome
2. Creates a new notebook
3. Pastes the full briefing text as a source
4. Triggers the **Audio Overview** generation
5. Polls for completion (~8-10 minutes)
6. Captures the notebook URL

### Step 6: Link the Audio Back to Notion

The Notion page gets updated with an audio link at the very top:

> **Don't feel like reading? Lera's got you covered.**

One click, and you're listening to a podcast-style summary of your entire briefing.

### Step 7: Finalize and Publish

The page status flips to **Published**. Properties are confirmed — date, run number, time slot, and tags all set. Done.

## Technical Discoveries and Gotchas

This is the section that will save you hours. I learned these the hard way.

### Notion MCP Page Creation — The Duplicate Pattern

The `notion-create-pages` tool has strict limitations that aren't immediately obvious:

- It does **NOT** accept `parent`, `database_id`, `data_source_id`, or `url` as parent keys
- Properties must be **flat primitives** — `string`, `number`, or `null` only
- No nested Notion API objects allowed

**The workaround:** Use `notion-duplicate-page` on an existing briefing page, then update properties and replace content separately. This is the reliable pattern for creating new database entries programmatically.

### Notion Update Commands — Know the Difference

These look similar but behave very differently:

| Command | Parameter | Format |
|---------|-----------|--------|
| `replace_content` | `new_str` | Single string replaces all content |
| `update_content` | `content_updates` | Array: `[{"old_str": "...", "new_str": "..."}]` |

Additional constraints:
- **Multi-select Tags** cannot be set via `update_properties` — they must be inherited from the duplicated template page
- The **icon** property also cannot be set via `update_properties`

### NotebookLM Browser Automation — React Textarea Injection

Standard DOM manipulation (`element.value = "text"`) doesn't work on NotebookLM's React-based textareas. React maintains its own internal state, so you need to use the native setter to bypass it:

```javascript
const nativeInputValueSetter = Object.getOwnPropertyDescriptor(
  window.HTMLTextAreaElement.prototype, 'value'
).set;

nativeInputValueSetter.call(textarea, text);
textarea.dispatchEvent(new Event('input', { bubbles: true }));
```

This triggers React's synthetic event system properly. Without this, the textarea looks filled but React's state remains empty and the submit button stays disabled.

### Audio Generation Timing

NotebookLM's Audio Overview generation takes **8-10 minutes**. Don't use a blind `sleep()` — poll for completion. The generation status is visible in the DOM, so you can check periodically and proceed as soon as it's ready.

### Running Unattended Overnight (macOS)

```bash
caffeinate -i &
```

This prevents macOS from sleeping. Key considerations:

- **API-based steps** (Notion MCP, web search) work fine with a locked screen
- **Chrome automation steps** require the machine to be awake
- **If Chrome automation fails**, the briefing content is still fully delivered to Notion — the audio link is the only thing that won't be attached

Leave Chrome open before bed. The pipeline handles the rest.

## The Persona: Lera

The briefing isn't just a data dump. It's delivered through **Lera** — a persona built into the skill definition:

- **Confident but not corporate** — she leads with what matters
- **Concise but not robotic** — every section intro has personality
- **Dry wit in the intros** — just enough to make scanning the briefing enjoyable
- **Forward-thinking** — she connects dots between headlines and flags what to watch

Lera isn't a gimmick. The persona constraint actually improves the output quality. It forces the model to synthesize rather than regurgitate, and to prioritize signal over noise.

## How to Replicate This

Here's what you need to build your own:

### 1. Set Up Claude Cowork

Install Claude Cowork (the desktop app). Connect:
- **Notion MCP** — for reading/writing to your Notion workspace
- **Claude-in-Chrome MCP** — for browser automation (NotebookLM)

### 2. Create the Notion Database

Create a database with these properties:

| Property | Type |
|----------|------|
| Title | Title |
| Run Number | Number (integer) |
| Status | Select (Draft, Published) |
| Time Slot | Rich text |
| Date | Date |
| Tags | Multi-select (Weather, World News, Tech, Claude AI, Claude Code, Cybersecurity, Hashtags, K-12 Education) |

Create one template entry manually. This becomes your duplication source.

### 3. Write the SKILL.md

Create a skill definition that describes the full pipeline. Include:

- Your **Notion database ID** (grab it from the database URL)
- Your **ZIP code** for weather lookups
- Your **topic preferences** for each of the 8 sections
- The **persona definition** for your briefing assistant
- The **step-by-step pipeline** so the model follows it deterministically

The skill definition is the blueprint. The more specific you are, the more consistent each run will be.

### 4. Schedule It

Use Cowork's scheduled tasks feature with a cron expression. For example:

```
# Run at 6:00 AM, 12:00 PM, and 6:00 PM daily
0 6,12,18 * * *
```

### 5. Prep for Overnight Runs

```bash
# Prevent macOS sleep
caffeinate -i &

# Leave Chrome open
# Leave Cowork running
```

That's it. Wake up to a fresh briefing.
